JGS Scripting Guide

JGS (JGraph Script) is a small array language for reading data, doing analysis, and producing plots inside JGraph. This guide is written for an engineer who has never seen JGraph before.

What is JGS?

JGraph is a desktop plotting and figure-editing application. JGS is one of its scripting languages — a purpose-built language (alongside the C# and Python engines) designed to feel familiar to anyone who has used an engineering array language. You write a .jgs file, run it, and it builds figures and prints results.

A JGS script is a sequence of statements executed top to bottom. It does three kinds of work:

  • Read data — load CSV/Excel tables from disk.
  • Compute — array math, statistics, string handling, filtering by condition.
  • Plot — issue stateful plotting commands (plot, scatter, subplot, title, …) that draw into a figure, then show it in a window.
Mental model: if you know NumPy or any engineering array language, you already know 90% of JGS. Arrays are first-class, indexing is by logical mask or by position, and plotting commands act on a single "current figure / current axes" that you switch with figure and subplot.

Running a script

Open the Script Workspace in JGraph, create or open a .jgs file, and run it (the editor gives you syntax highlighting, autocomplete, and a debugger). Plotting commands accumulate into the current figure; show() opens (or updates) a window for it at any point, and — since M21 — any figures the script created but never show()ed are displayed automatically when the run finishes. Re-running a script reuses the same figure windows by number rather than piling up new ones. A script can also be run from the command line, with no user interface at all.

A script's only access to the outside world is the table readers (readcsv, readxlsx, readtable), the image readers (imread, imwrite), the figure-file commands (savefigure, loadfigure, exportfigure), and run for including another script. There is no arbitrary file, network, or system access.

Everything a figure contains is also editable after the script has drawn it, by hand: select an object in the plot browser and edit it in the property inspector — down to a title's font, size, weight and colour. The legend is a first-class object whose entries you can rename, reorder, and hide, and which you can drag anywhere on the plot; missing pieces (titles, a legend, a colorbar, extra axes, subplot grids, annotations) are added from the browser's context menu or its Add ▾ button. Save the result with savefigure, or from the figure window, and it reopens exactly as you left it.

From the command line

jgraph.exe runs a script without opening JGraph at all. There is no window, no user interface, and no display is required — so a script can run over a remote session, from a build step, or on a schedule. It writes to standard output, and returns an exit code the shell can act on.

jgraph -batch "let x = 1:10; disp(sum(x))"       # prints 55, then exits
jgraph -batch "analysis.jgs" -logfile run.log     # run a file, tee everything to a log
jgraph -r "plot(1:10)"                            # run, then leave the session open
jgraph -help                                     # the flag reference, and this guide
-batch "statement"Run without a user interface, log to standard output, and exit when the script finishes.
-r "statement"Run the statement in the interactive application and keep the session open, so you can carry on from where the script left off. The application closes when a script calls exit.
-logfile "path"Also write all output, results, and errors to a text file. The file is appended to and flushed after every line, so it survives an interrupted run.
-showfiguresWith -batch only: display figures in standalone windows (there is no main window) instead of suppressing them. The process then exits once the last window is closed.
-sd "directory"Use this directory instead of the shell's current one for relative paths.
-h, -helpPrint the startup options, and open this guide.

The statement is JGS source unless it names a file that exists, in which case that file is run and its extension picks the language (.jgs, .csx/.cs, .py). Relative paths — including where exportfigure writes — resolve against the shell's current directory; a script run by path can still read data files sitting beside it.

Figures under -batch: there is nowhere to show them, so show() reports the figure instead of displaying it. Produce output with exportfigure (PNG/SVG/PDF/…) or savefigure — both work perfectly well with no display — or pass -showfigures when you do want windows.

Exit codes: 0 the script finished · 1 the script failed · 2 the command line was invalid · n whatever the script passed to exit(n). That last one is how a script signals a result to a build or a scheduled job:

let t = readcsv("batch_yield.csv")
let yield = mean(column(t, "pass"))
exportfigure("yield.png")
if yield < 0.95 {
    disp("Yield below target")
    exit(1)          # the build step fails
}

Syntax basics

let x = 5Declare a variable. Reassign later with x = 6 (no let).
x += 2   x++Compound assignment (+= -= *= /= %=) and increment/decrement (x++, ++x, x--, --x) — full C semantics, also on array elements (a[i] += 1).
let [X, Y] = pair()Destructuring: bind the elements of a 2-element (or n-element) array to names — how you receive meshgrid's two matrices.
newline or ;Statements are separated by line breaks or semicolons. Blank lines are ignored. A trailing ; additionally suppresses console echo — see array-language scripting.
# ... or // ...Line comment to end of line. (There are no block comments.)
"text" / 'text'Strings — double or single quotes are interchangeable. Escapes: \n \t \r \\ \" \'.
[1, 2, 3]Array literal. May span multiple lines (line breaks inside [ ] and ( ) are not statement separators).

Identifiers start with a letter or _ and continue with letters, digits, or _. Numbers are decimal and may use scientific notation (1.5e-3).

# A first script
let x = linspace(0, 6.28, 100)   # 100 points, 0 .. 2π
let y = sin(x)                   # element-wise over the array
plot(x, y, "r-")               # red solid line
title("A sine wave")
show()                            # open it in a window

Operators

Arithmetic operators broadcast element-wise over arrays, exactly like NumPy: a scalar applies to every element, and two equal-length arrays combine pairwise.

OperatorsMeaningOn arrays
+ - * / %Add, subtract, multiply, divide, remainderElement-wise; scalar broadcasts. [1,2,3] * 2 → [2,4,6].
< <= > >=ComparisonsElement-wise when either side is an array → a bool array (mask). temp > 85.
== !=Equal / not equalElement-wise when either side is an array (works on string arrays too, e.g. ids == "ABC"). Scalar-to-scalar gives one bool.
&& ||Logical AND / OR (short-circuit)Scalar only, on whole-value truthiness. For element-wise logic use and/or/not.
!   -Unary NOT / negate! negates a bool; - negates a number.
= += -= *= /= %=Assignment (plain and compound)An expression yielding the stored value: a = b = 0 chains, let y = (x += 2) works. Compound forms broadcast like their operator: xs += 1 is elementwise.
++ --Increment / decrement by 1Postfix (x++) yields the old value; prefix (++x) the new. Works on variables and array elements; the target must hold a number.
^   .^PowerElement-wise; binds tighter than unary minus (-2^2 → -4), associates left (2^3^2 → 64), and allows 2^-3.
~   ~=   .*   ./Array-language spellingsAliases of !, !=, *, / — JGS operators already broadcast element-wise, so a .* b and a * b mean the same thing.
a:b   a:step:bColon rangeAn inclusive sequence: 2:6 → [2,3,4,5,6], 0:1/fs:3 → 3001 samples. Looser than arithmetic, tighter than comparisons.

Precedence, high to low: ^ → unary → * / % → + - → ranges (:) → comparisons → == != → && → || → assignment (right-associative). Parenthesize with ( ) as usual.

Two C-style footguns to know: (1) since assignment is an expression, if x = 5 { } is legal and assigns — write == to compare (assigning to an undeclared name is still an error, which catches most slips). (2) 5--3 no longer parses because -- is now the decrement operator — write 5 - -3.
Truthiness: an array is truthy only if it is non-empty and every element is truthy. So if temp > 85 { ... } runs only when all readings exceed 85. To ask "any?" or "how many?", use any(mask), all(mask), or sum(mask). To test emptiness, use length(a) > 0.

Array-language scripting

Since M21, scripts written for other engineering array languages run in JGS with only minimal edits (convert % comments to //, add let on first assignments, add commas inside array literals). The pieces:

x = 5 vs x = 5;Echo & suppression. A statement without a trailing ; echoes its result to the console (x = 5); with ; it stays silent. A bare expression binds ans (even when suppressed) and echoes ans = …; a bare variable name displays it. Long arrays truncate with an element count.
for k = 2:n … endend-terminated blocks. for/while/if/fn bodies may be closed by end instead of braces; elseif chains share one end. Both styles coexist. end and elseif are reserved words.
t = 0:1/fs:3Colon ranges, inclusive with an endpoint tolerance (that one is exactly 3001 samples).
[a; b]Rows & vertical concat. [1, 2; 3, 4] is a matrix; when rows contain arrays the whole thing flattens into one array — [audio; zeros(k, 1)] pads a signal.
figure;Command form & auto-display. A bare builtin name as a statement calls it with no arguments. And every figure a script creates is shown automatically when the run finishes — no show() needed (explicit show() still works and is not repeated).
3 + 4iComplex numbers. Literals 2i/1.5j; + - * / ^ promote; abs is magnitude, plus real, imag, conj, angle. fft returns complex bins. Ordering comparisons and plotting complex data are errors that tell you to take abs()/real() first. Zero-imaginary results collapse back to plain numbers.
pi e inf nanBuilt-in constants.
Deliberate differences from the languages JGS borrows from: % is still the modulo operator, not a comment (use // or #; mod(a,b) exists); let is still required for a first binding; array literals need commas ([0 20 30] is an error); and arrays assign by reference — X2 = X1; X2(1) = 0 also changes X1.

Arrays & indexing

Arrays are the workhorse. Build them with a literal, a colon range (2:6), or a constructor (linspace, range, zeros, ones, rand), then transform them with math and the array builtins.

Indexing is 0-based, in two interchangeable spellings. a[i] and a(i) mean exactly the same thing and both support end, colon slices, masks, gathers, and slice assignment:

let a = [10, 20, 30, 40]
a[0]          # 10  — the first element
a(0)          # 10  — same thing, other spelling
a[end]        # 40  — 'end' is the last index, i.e. length - 1
a(1:end)      # [20, 30, 40] — colon slice
a[:]          # everything, as a copy
a(1) = 99     # element write → [10, 99, 30, 40]
a[0:1] = 0    # slice write, scalar broadcasts → [0, 0, 30, 40]
length(a)     # 4

Both forms also accept an array of indices to gather several elements at once, and strings can be indexed the same way (you get back a string):

a([0, 3])   # [10, 40]  — a gather, pairs with find()
The one difference between the spellings: f(x) calls a function, f[x] does not — so brackets are the unambiguous way to say "index this". There is no member access (obj.field). Colon slices work only inside an index — elsewhere a:b is a free-standing range value.
Porting from MATLAB? Every index drops by one: x(1) becomes x(0), x(1:n) becomes x(0:n-1), and a for k = 2:N accumulator loop becomes for k = 1:N-1. Expressions written against end usually survive untouched, because end shifts down with them. find(mask, 1) restores MATLAB numbering where you would rather not rewrite the call site.

Masks & logical indexing

This is the heart of JGS analysis. A comparison over an array yields a mask — a bool array of the same length. Index an array with a mask to keep only the elements where the mask is true:

let temp = column(t, "Temperature")
let hotOnes = temp(temp > 85)   # keep readings above 85 (paren or bracket form)

Masks align across columns, which is what makes "filter the whole dataset by a condition" work. find turns a mask into the matching row indices, and those indices map back to every column of the original data — row 3 of one derived array is the same record as row 3 of another:

let hot     = find(temp > 85)   # row numbers (0-based) where the condition holds
let hotIds  = ids(hot)              # the serial numbers of those same rows
let hotVolt = volt(hot)              # and their voltages — all still aligned
find returns 0-based indices, so its result gathers directly in either spelling — volt(hot) and volt[hot] are the same. Pass a base of 1 (find(mask, 1)) when you want MATLAB's numbering for a report or a ported script.
Index valueBehavior
a single numberReturns that one element (0-based, in either spelling).
a bool array (mask)Filters — keeps elements where the mask is true. The mask length must equal the array length.
a number arrayGathers — returns those elements in that order (great with find).

Matrices

A matrix in JGS is simply an array of equal-length row arrays — there is no separate type. Build one with a nested literal, zeros(rows, cols) / ones(rows, cols), or meshgrid; index it as m[row][col].

let m = [[1, 2], [3, 4]]
m[1][0]                       # 3 — row 1, column 0
let [X, Y] = meshgrid([1,2,3], [10,20])   # coordinate matrices (2 rows x 3 cols)

Arithmetic operators and the element-wise math builtins recurse into nested arrays, so the classic surface recipe works verbatim:

let x = linspace(-8, 8, 60)
let [X, Y] = meshgrid(x, x)
let R = sqrt(X * X + Y * Y) + 0.01
surf(x, x, sin(R) / R)         # the sinc "sombrero"
Limits: comparisons/masks and the reductions (sum, mean, …) operate on flat arrays only — apply them per row (sum(m[0])) if needed. Matrix arguments to plotting verbs must be rectangular; ragged rows are an error.

Control flow

if / else if / else

if mean(v) > 3.3 {
    print("high")
} else if mean(v) > 3.0 {
    print("nominal")
} else {
    print("low")
}

while

let i = 0
while i < length(devices) {
    print(devices[i])
    i = i + 1
}

for … in

for id in devices {
    print("device:", id)
}

break / continue

for v in readings {
    if isnan(v) { continue }
    if v > limit { break }
}

Blocks are always delimited by { }. The condition is not parenthesized (though you may add parens as grouping). break and continue work inside while and for.

Defining functions

Define reusable functions with fn. Use return to produce a value; a function with no return yields null.

fn rms(a) {
    return sqrt(mean(a * a))
}

let r = rms([3, 4, 0])   # 2.886...

Brace placement is flexible: the { may sit on its own line (after fn, if, for, while, and else), and a function's parameter list may follow its name on the next line. To return several values, return an array and destructure it: return [slope, offset] … let [m, b] = fit(x, y).

Figures, axes & show

JGS follows the classic stateful plotting model. There is always a current figure and, within it, a current axes. Plotting verbs (plot, scatter, bar, …) draw into the current axes; label verbs (title, xlabel, legend, …) modify it.

figure()Start a new figure and make it current. Returns its number (1, 2, 3, …).
figure(2)Select figure 2 (creating it if needed) and make it current — the way to switch between figures.
subplot(2,1,1)Split the figure into a 2×1 grid and select cell 1 (a grid cell number, so it starts at 1, row-major) as the current axes.
hold(true)Keep existing series when plotting more, so multiple plot calls overlay on one axes. hold(false) replaces.
show()Open (or refresh) a window for the current figure. show(2) shows figure 2.

Each figure opens in its own window with the full JGraph chrome — pan, zoom, the property inspector, and export all work just as in the main app. Nothing appears until you call show; re-running the script updates the same numbered windows in place instead of spawning new ones.

Line-spec strings

plot and the semilog*/loglog verbs take an optional spec string combining a color, a line style, and a marker (any order; unknown characters are ignored). Example: "r--o" = red, dashed line, circle markers.

Colors

b blueg green
r redc cyan
m magentay yellow
k blackw white

Line styles & markers

- solido circle   . point
-- dashedx cross   + plus
: dotted* star   s square
-. dash-dotd diamond   ^ v triangles

If the spec names a marker but no line style (e.g. "ro"), you get markers only — no connecting line. Include a line style ("-o") to draw both.

File paths

Relative paths passed to readcsv, savefigure, etc. resolve against the script's own folder first, then the workspace root. Absolute paths are used as-is. New files written by savefigure/exportfigure land in the working directory (the script's folder / workspace root), not wherever the app happens to be running.

3D surfaces, contours & colormaps

Surface verbs switch the current axes into 3D mode: it renders as a rotatable coordinate box. Drag the plot to rotate the camera, use the mouse wheel to zoom in and out, and undo/redo to step the view back. z arguments are matrices; x/y are grid vectors matching its columns/rows.

# The classic "sombrero", rotatable with the mouse
let x = linspace(-8, 8, 60)
let [X, Y] = meshgrid(x, x)
let R = sqrt(X * X + Y * Y) + 0.01
surf(x, x, sin(R) / R)
colormap("jet")
colorbar()
view(-37.5, 30)
title("sinc(r)")
show()
Practical size cap: surfaces re-project and depth-sort every cell while you drag; grids up to ~150×150 stay smooth.

DSP & audio

The signal-processing verbs (M21). The FFT handles any length in O(n log n) (Bluestein's algorithm for non-powers-of-two), so a raw audio clip transforms directly. Frequencies are normalized to Nyquist (0..1) where noted.

SignatureDescription
fft(x, n) · ifft(x, n)Forward / inverse discrete Fourier transform (complex result; optional length pads or truncates). real(ifft(X)) recovers a real signal.
fftshift(x) · ifftshift(x)Rotate a spectrum so DC sits at the center, and back.
filter(b, a, x)Apply the digital filter b/a to signal x (Direct Form II transposed, zero initial state). a may be the scalar 1 for FIR.
freqz(b, a, count, fs)Frequency response: let [H, f] = freqz(h, 1, 1024, fs) — complex H plus the frequency axis (fs defaults to 2 → normalized f). Plot abs(H).

Time & date

A stopwatch and date functions. Used as bare statements, tic; starts the timer and toc reports the elapsed seconds; inside an expression call them with parentheses (toc()). Dates are serial date numbers — days since year 0, where datenum(1970, 1, 1) is 719529 and the fractional part is the time of day. There is no dedicated date type: functions return plain numbers, [y m d h mi s] vectors, or strings.

Large arrays & performance

Since M22, numeric arrays are stored packed — 8 bytes per element in flat buffers with hardware-SIMD elementwise math — so million-sample DSP scripts run in a fraction of the memory and time. This is transparent: semantics, aliasing, indexing, and results are identical. The practical notes:

  • Arrays longer than 1000 elements print as a short prefix plus a count ([0, 0.001, …] (3001 elements)) instead of every value.
  • A colon range may produce up to 250 million elements (2 GB); very large arrays automatically move to native memory or, when RAM is tight, an SSD-backed temporary file — slower, but no out-of-memory crash.
  • Double-clicking a variable over 2 million elements shows a status message instead of the data viewer; index a smaller slice to inspect it.
  • Stop interrupts even a single giant elementwise operation mid-flight.

Worked example — data analysis

A test log where the real table sits below six lines of junk preamble. This parses it, computes per-device statistics, isolates the over-temperature readings across all columns, and plots the result.

# Skip 6 preamble lines (tester name, date, rig info) before the header row
let t = readcsv("test_log.csv", 6)
print("columns:", join(colnames(t), ", "), " rows:", rowcount(t))

let ids  = textcolumn(t, "SerialNumber")   # IDs stay as strings
let temp = column(t, "Temperature")
let volt = column(t, "Voltage")

# Per-device summary — build the unique ID list, then filter by string mask
let devices = unique(ids)
for id in devices {
    let mask  = ids == id            # element-wise string compare → mask
    let temps = temp(mask)           # logical indexing keeps that device's rows
    print(sprintf("%s: n=%d mean=%.2f std=%.2f p95=%.1f",
        id, sum(mask), mean(temps), std(temps), percentile(temps, 95)))
}

# Over-temp records — find() gives row indices that map back to EVERY column
let hot     = find(temp > 85)
let hotIds  = ids(hot)
let hotVolt = volt(hot)
print("over-temp readings:", length(hot), " first id:", hotIds(1))

figure()
scatter(hotVolt, temp(hot))
title("Voltage vs Temperature (over 85 C)")
xlabel("Voltage")
ylabel("Temperature")
grid(true)
show()
